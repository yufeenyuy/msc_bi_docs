### Unit 1: Motivation and Challenges of IT Governance

The section is explained based on a case study related to the breach suffered by UBER around November 14th 2016.
The leakage came as a result of using Amazon Web Service resources with a program storec on GitHub that used hard 
coded *secrete key* to retrieve information. Code wrote and stored on Github is generally accessible so hackers
used these secrete keys to compromise UBER's Systems and threatened to publish this data in the public if a 
ransom was not paid. According to the hackers, by the time they contacted the chief security officer at UBER,
they had released part of the data in the **dark web** already. The *Dark Web* is the part of the internet that
is not accessible via traditional search engines like *google, duck-duck-go* and thus requires more details before
it can be accessed. Dark web consist of **legitimate and illegal buying and selling activities**. An example of
an illegal activity on the dark web is the *sell of stolen personal data*. The hackers claimed that they had
access to classical UBER Data including: Workers names, date of births, addresses, customer credit card details,
drivers license information of UBER workers etc.

Using the above description the chief executive officer had to take action to resolve this issue before UBER would
be released to the entire public. There are a few questions that the Chief Security Officer had to answer to resolve
this issue and avoid similar scenarios to occure in the future.
